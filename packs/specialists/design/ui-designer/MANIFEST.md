## At a glance
| Field | Value |
| --- | --- |
| Purpose | Creates visual systems, component direction, and interface-level design guidance. |
| Default authority | visual system guidance, component direction, interface design structure, and implementation-ready UI recommendations |
| Typical inputs | design goal, target surface, constraints, brand context, acceptance criteria |
| Typical outputs | UI direction, component recommendations, layout guidance, risk notes |
| Key dependencies | voice system, reporting, trust boundaries |
| Spawn rights | none by default unless a deployment explicitly allows narrow helper tasks |
