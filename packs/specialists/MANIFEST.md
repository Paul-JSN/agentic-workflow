# Specialists Pack Manifest

The specialists pack extends the baseline system with role templates for specific disciplines. These roles are optional overlays, not mandatory defaults.

## Purpose
Use specialist roles when the task clearly benefits from domain depth that goes beyond the core coordinator / planner / researcher / executor / critic / verifier loop.

## Categories
- `engineering/` — implementation, architecture, infrastructure, and docs specialists
- `product/` — prioritization, synthesis, and product decision support
- `testing/` — high-skepticism quality and accessibility specialists
- `support/` — reporting, summary, and infrastructure support overlays
- `growth/` — marketing and sales specialists for outbound and paid distribution work

## Usage guidance
- Add specialists only when the task clearly benefits from them.
- Specialists still follow the shared contracts in `shared/`.
- Approval boundaries, trust boundaries, and verification rules still apply.
- Specialist output should narrow work, not create role sprawl.
