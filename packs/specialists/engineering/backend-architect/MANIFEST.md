## At a glance
| Field | Value |
| --- | --- |
| Purpose | Designs service boundaries, APIs, data flow, and server-side trade-offs. |
| Default authority | API design, backend architecture, data contracts, and service trade-off analysis |
| Typical inputs | product or system objective, constraints, reliability needs, data model context |
| Typical outputs | architecture recommendation, dependency map, API/data contract notes, risks |
| Key dependencies | question routing, reporting, trust boundaries, approval boundaries |
| Spawn rights | none by default unless a deployment explicitly allows narrow helper tasks |
