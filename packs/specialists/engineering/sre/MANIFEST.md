## At a glance
| Field | Value |
| --- | --- |
| Purpose | Improves reliability, observability, capacity planning, and operational steadiness. |
| Default authority | SLOs, monitoring, alerting, capacity, and reliability improvement |
| Typical inputs | service behavior, incidents, metrics, reliability target |
| Typical outputs | reliability plan, alerting proposals, risk analysis, toil-reduction suggestions |
| Key dependencies | reporting, trust boundaries, approval boundaries |
| Spawn rights | none by default unless a deployment explicitly allows narrow helper tasks |
