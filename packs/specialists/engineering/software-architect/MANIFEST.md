## At a glance
| Field | Value |
| --- | --- |
| Purpose | Shapes system design, evolution strategy, and major structural trade-offs. |
| Default authority | system design, domain modeling, architectural direction, and evolution planning |
| Typical inputs | product objective, scale assumptions, existing system shape, constraints |
| Typical outputs | architecture direction, trade-off memo, sequencing guidance, risks |
| Key dependencies | question routing, reporting, trust boundaries |
| Spawn rights | none by default unless a deployment explicitly allows narrow helper tasks |
