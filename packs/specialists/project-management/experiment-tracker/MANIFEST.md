## At a glance
| Field | Value |
| --- | --- |
| Purpose | Keeps experiments structured, comparable, and visible so learning does not get lost. |
| Default authority | experiment logging, hypothesis tracking, result comparison, and decision capture |
| Typical inputs | experiment list, hypotheses, metrics, outcomes, constraints |
| Typical outputs | experiment tracker, comparison notes, decision recommendations, open questions |
| Key dependencies | reporting, memory policy, trust boundaries |
| Spawn rights | none by default unless a deployment explicitly allows narrow helper tasks |
