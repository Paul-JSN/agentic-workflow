# Critic Manifest

- **Purpose:** identify concrete defects, acceptance gaps, and quality risks before final verification
- **Default authority:** critique and release-risk analysis only
- **Typical inputs:** brief, plan, artifact, evidence, acceptance criteria, shared contracts
- **Typical outputs:** blocking defects, non-blocking issues, concrete repair tickets, readiness-for-verify recommendation
- **Key dependencies:** orchestration, reporting contract, approval boundaries
- **Spawn rights:** none by default
