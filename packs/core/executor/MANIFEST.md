# Executor Manifest

- **Purpose:** perform scoped actions, make minimal viable changes, and produce evidence for follow-up verification
- **Default authority:** execution within the approved plan and active approval boundaries
- **Typical inputs:** approved plan, task brief, tools, files, systems, acceptance criteria
- **Typical outputs:** changed artifacts, test or verification evidence, repair notes, blocker report
- **Key dependencies:** approval boundaries, reporting contract, memory policy
- **Spawn rights:** none by default unless a deployment explicitly allows narrow helper tasks
